The goal is to create a local-first Retrieval-Augmented Generation app that lets me search, summarize, compare, and ask questions about my contracts and business documents. The app should behave like ChatGPT, but it must only answer using my indexed document library.

Build this in phases. Start with a working MVP first, then structure the code so advanced features can be added later.

PHASE 1: LOCAL RAG MVP

Create a desktop app for Windows with the following stack:

Frontend:
- React
- TypeScript
- Tailwind CSS

Desktop framework:
- Tauri preferred
- Electron acceptable if Tauri is not practical

Backend:
- Python
- FastAPI

Database:
- SQLite for metadata, conversations, document records, and settings

Vector database:
- ChromaDB

Document processing:
- PyMuPDF for PDFs
- python-docx for Word files
- openpyxl or pandas for Excel files
- markdown/text parsers for Markdown and text
- BeautifulSoup for HTML
- python-pptx for PowerPoint
- pytesseract for OCR on scanned PDFs where possible

Embeddings:
- Default local embedding model if possible
- Support OpenAI embeddings as an optional setting later
- Structure the embedding provider so it can be swapped

LLM support:
- Ollama for local models
- Optional OpenAI
- Optional Anthropic Claude
- The user should be able to choose the model provider in Settings

- Split content into semantic chunks

RAG ANSWERING RULES

The assistant must follow these rules:

- Only answer using indexed document content
- Never fabricate information
- If the answer cannot be found, respond exactly:
  “I could not locate that information in the indexed documents.”
- Every answer should include citations when possible
- Citations should include:
  - Document name
  - Page number if available
  - Section heading if available
  - Relevant quoted text
  - Confidence score

Example questions the app should support:

- “What is the termination clause in the Microsoft contract?”
- “What is the payment schedule?”
- “Does Vendor A require cyber insurance?”
- “What contracts expire this year?”
- “Which contracts contain indemnification language?”
- “Which documents reference SharePoint?”
- “Which SOW mentions Azure?”

Security is very important.

The app should be local-first.

Requirements:

- Documents remain on my computer by default
- No telemetry
- No analytics
- No external uploads unless I explicitly enable cloud AI
- Show a clear warning when cloud AI is enabled
- Store API keys securely where possible
- Do not log full document contents
- Design the vector database so encryption can be added
- Include a Local Only mode
